# 使い方

このサイトが提供する機能は3つです。UXに絞って説明します。

## 1. 抽選結果を記録する画面
- 武器/属性を選んでテーブルを決める
- シリーズスキル/グループスキルを選択して行を追加する
- お気に入りは行にチェックが付くのみで、並び順はカーソル降順
  - 同一カーソル内では登録日時の降順
- カーソルが通過済みの行はデフォルト非表示で、表示ボタンで切り替えられる
- 記録済みの行はプルダウンでシリーズ/グループの編集ができる
- 表示スキル設定で選択肢の表示/非表示を制御できる
  - 非表示の選択肢は「非表示スキル」として記録される

## 2. カーソルを進める画面
- カーソルは全属性/全武器で共通
- 武器/属性でフィルターできる
- フィルター後、そのカーソルで取得できる抽選結果のうち NULL でないものをすべて表示する
- この一覧ではお気に入りを優先表示する
- その中から1つを選ぶとカーソルが進み、次の候補が表示される
- カーソルが進むとき、カーソル位置がまだ NULL のテーブルは
  シリーズスキル/グループスキルともに「不明」と表示する
- 「不明」は `docs/group_skills.yaml` / `docs/series_skills.yaml` の末尾にある選択肢で、
  通常の選択肢と同じ扱いにする

## 3. 記録内容を確認する画面
- 武器/属性で表示対象を絞り込む
- 同一カーソルの結果を1行にまとめて表示する
  - 列は保存済みの `武器::属性` のパターンごとに並ぶ
  - セルに「お気に入り/シリーズ/グループ」をブロック表示する
- セルをクリックするとお気に入りのON/OFFを切り替えられる
- JSONのエクスポート/インポートでバックアップができる

## NULL の定義
ローカルストレージに保存されたテーブルごとのJSONデータで、
カーソルに該当するIDを持つレコードが存在しない場合を NULL とする。

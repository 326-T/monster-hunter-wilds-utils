# OCR 実験機能 仕様

## 目的
記録画面でゲーム画面をOCRし、抽選結果の登録を補助する。

## UX フロー
1. ウェブカメラを起動
2. ユーザが ROI (読み取り範囲) をドラッグで指定
3. シャッターを押すと OCR を実行
4. 読み取り結果からシリーズ/グループを一致判定し、そのまま登録

## 登録ルール
- OCR テキストに既存のスキル名が含まれる場合、最長一致を採用
- シリーズ/グループが見つからない場合は「不明」を登録

## 技術要件
- クライアントサイドのみで完結
- `tesseract.js` を使用（軽量OCR）
- WebGPU が使える場合は、前処理（グレースケール/二値化）をGPUで実行
  - WebGPU 非対応時は CPU で同等処理
- 2値化や縮小などで計算負荷を抑える

## パフォーマンス最適化
- ROI のみ切り出して処理
- 最大幅を抑えて縮小（過剰な解像度は避ける）
- 2値化して文字のコントラストを強調

## 既存機能への影響
- OCR は独立コンポーネント/フックとして切り出す
- 記録画面に追加し、既存の保存フローは変更しない
